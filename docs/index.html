Local Ollama models in Cursor IDE

cursor-ollama exposes your local Ollama through a secure Cloudflare Tunnel, rewrites model names for Cursor's allowlist, and validates Bearer auth — one command to set up.

Install from npm View on GitHub
# Quick start
$ npm i -g cursor-ollama
$ cursor-ollama init
$ cursor-ollama setup
$ cursor-ollama cursor-config

# Cursor Settings → Models
Base URL: https://your-tunnel.example.com/v1
Add model: gpt-4o-mini → Ollama: qwen2.5-coder:7b

Why cursor-ollama?

Cursor-compatible routing

Cursor can't reach localhost. Cloudflare Tunnel gives a public HTTPS endpoint Cursor's backend can call.

Model name rewrite

Use allowlisted names like gpt-4o-mini in Cursor while Ollama runs qwen2.5-coder:7b behind the proxy.

Bearer auth built in

Strategy C proxy validates your API key before forwarding — no bare Ollama exposed to the internet.

Cross-platform CLI

Works on Windows, macOS, and Linux. Init wizard, doctor checks, verify, and OS service install.

Zero OpenAI calls

BYOK routes chat through your tunnel. You are not calling OpenAI — the model name is just a label Cursor accepts.

Open source

MIT licensed. Full test suite, CI matrix, and documented fallback strategies.

How it works

Cursor cloud → HTTPS tunnel → Proxy :11435 → Ollama :11434

The proxy rewrites model names and validates Bearer tokens. cloudflared forwards traffic to the proxy, never directly to Ollama.

Install

npm install -g cursor-ollama
cursor-ollama doctor
cursor-ollama init
cloudflared tunnel login
cloudflared tunnel create cursor-ollama
cursor-ollama setup
cursor-ollama cursor-config

Requires Node.js 18+, Ollama, cloudflared, and a domain on Cloudflare DNS.
